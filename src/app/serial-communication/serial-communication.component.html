Serial Communications

RS-232

  • Transmits binary data on a wire where:
    • +5 to +12 volts = binary value 0
    • -5 to -12 volts = binary value 1
  • Uses a UART transmitter to transmit the signals.
  • Three wires are used:
    • Tx: Used for transmitting data
    • Rx: Used for receiving data
    • GND: 0V ground reference for Tx and Rx
  • A BAUD rate (bits/s) is used to keep track of each pulse.
  • The last bit is sometimes used as a parity bit.
    • The parity bit helps determine if data was corrupted during transfer

RS-485

  • Transmits binary data on two twisted wires:
    • The first wire is named D+
    • The second wire is named D-
    • Binary values are represented by a voltage differential between the two wires.
    • ([D+] - [D-]) < -200mV represents a binary value of 1
    • ([D+] - [D-]) > +200mV represents a binary value of 0
  • Also uses a UART transmitter to transmit signals.
  • More immune to electrical noise than RS-232
  • Faster than RS-232
  • Why is RS-485 more immune to electrical noise than RS-232?
    • The twisted wires mean electrical noise effects the D+ and D- signals equally.
    • Since the data is dependent a voltage differential, and electrical noise effects both wires equally, The differential voltage is not affected. Each line may change several volts with respect to ground, but the difference between the two remains the same.
  • Why is RS-485 faster than RS-232 if they both use UART Transmitters?
    • RS-485 has a lower voltage swing between 0 and 1. RS-232 needs to swing from -5 (or-12) to +5 (or +12) to change from 1 to 0. Whereas RS-485 only needs to swing from -200 mV to +200mV
